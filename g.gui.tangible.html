DESCRIPTION

Tangible Landscape is a collaborative modeling environment for analysis of terrain changes which couples a scanner, projector and a physical 3D model with GRASS GIS.

The software consists of a plugin for GRASS GIS and addon r.in.kinect for Kinect for Windows.

Scanning and analyses controls

Upper row of buttons controls calibration and scanning mode:
Start
Start continuous scanning and analyses
Stop
Stop continuous scanning and analyses
Calibrate
Scan and save calibration data, no analyses computed.
Scan once
Scan only once and then run analyses

Calibration

Calibration is a simple process ensuring the resulting scan is horizontal (not tiled). This can happen when the scanner is not aligned perfectly vertically. Calibration is performed by scanning (by pressing Calibrate button) an empty table before placing the model on the table. The calibration parameters are saved into GUI settings file in GRASS home folder and applied during scanning to compensate for the misalignment.

Scanning options

Scanning tab consists of several input fields influencing the georeferencing and quality of the resulting scanned raster. Most of these options are directly input into r.in.kinect module:
Name of scanned raster
Name of the raster derived from scanning
Reference DEM
Name of the reference DEM from which georeferencing information is taken. The scanned raster will have the same extent.
Reference region
Name of the reference region file from which georeferencing information is taken. The scanned raster will have the same extent.
Z-exaggeration
Specify the vertical exaggeration of the physical model to scale the digital model correctly to real world dimensions
Number of scans
Number of scans to integrate. More scans will result in slower scanning intervals, but reduced gaps in the resulting scanned raster.
Rotation angle
Rotate scan along Z axis in degrees counterclockwise
Smooth value
Set smoothing radius (in mm) of Moving Least Squares (MLS) surface reconstruction method
Resolution
Resolution of the scanned raster in mm
Limit scan vertically
Specify minimum and maximum distance from the scanner in cm, points outside of that limit are filtered
Trim scan
Specify horizontal distance in cm from the center of the scanner in N, S, E and W directions to limit scanned area
Trim tolerance
If specified, use automatic horizontal trimming to avoid hands and other objects lying around the model. Higher values close to 1 result in more aggressive trimming. Recommended only for rectangular models.
Use interpolation instead of binning
Interpolation can fill gaps in the raster, however it takes more time. Binning is fast but can leave some gaps in case of insufficient point cloud coverage.

Analyses

Analyses file
Path to a Python file with analyses to be run whenever a new scanned raster is processed. This files contains defined functions performing different geospatial tasks. All functions starting with run_ are launched, other ones are ignored. GRASS GIS modules are called and chained in these functions using GRASS Python Scripting API. This file is re-imported in every loop, so saved changes are applied for new loop. In case of error (for example syntax error), the file is ignored until the error is fixed.
Create new file
Creates a new analyses file in user defined location with pre-defined analyses which need to be uncommented if they are to be used. This file can then be edited and more analyses can be added.
All settings are saved when the dialog is closed to GRASS home folder and are used when re-opening the dialog.
